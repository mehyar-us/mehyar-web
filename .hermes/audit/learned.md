## 2026-07-09 · turn-016 (sha 6dd002f)
Add-on SEO wins belong in `scripts/route-jsonld.json`, not in TSX components. The injection script is idempotent (marker `data-route-jsonld=<route>`), validated by the build, and survives no-JS crawlers — whereas runtime `SeoManager.tsx` only fires on hydration. Cost of adding a new schema entry: ~3 min (one JSON entry, one build, one live curl). Lesson: any structured-data surface the SEO sheet still wants should go through the route-jsonld.json pipeline, not as inline `<Helmet>` or React-effect side-effects. Also: state.md drift — when the loop skips a tick-id bump (turn-015's rss feed landed without bumping state.md from 14), the next tick must reconcile. Always re-read state.md at the top of the tick and update the count even if "nothing changed semantically" — drift causes downstream confusion when the iteration diary and tick_id disagree.

CF Pages deploy lag observed today: ~5min 30s from `git push origin main` to `curl -sL https://mehyar.us/about/` showing the new bundle. Slightly longer than the 3-5min estimate in turn-014 — likely because /about isn't a hot path so CF may not warm the worker as eagerly. Plan a 6-minute sleep before the first live check on a cold route.

Also learned on this tick: patch tool truncated a multi-line new_string in copy-route-shells.mjs (a 20-line indented block was cut off mid-line), leaving the JS object broken. The heredoc approach is more debuggable and survives prompt truncation.
